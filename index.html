What is ActiveCanvas?

A CMS engine you mount into any Rails app.

ActiveCanvas is a Ruby gem that adds a full page management system to your existing Rails application. Mount it at any path — /pages, /cms, or even / to power your entire site. The engine provides an admin panel and serves published pages at your chosen path. No separate app needed.

Mountable Engine

Fully namespaced Rails engine. Mount it at any path. It won't conflict with your host app's models, routes, or assets.

Visual + Code Editor

GrapeJS-powered drag-and-drop editor with a built-in HTML/CSS code panel. Switch between visual and code editing freely.

AI Content Generation

Generate text, images, and convert screenshots to HTML. Supports OpenAI, Anthropic, and OpenRouter via the ruby_llm gem.

Capabilities

Everything you need
to manage pages.

Built as a complete solution — not a minimal scaffold you'll spend weeks extending.

CSS Framework Support

Tailwind CSS v4 with per-page compilation via tailwindcss-ruby, Bootstrap 5, or vanilla CSS. Switch frameworks in settings.

Page Versioning

Automatic version tracking on every save. View up to 50 versions per page with unified diffs, change summaries, and author tracking.

Media Management

Upload images and files via Active Storage. Signed URLs for private storage, automatic thumbnails, and file validation with magic byte checking.

Reusable Partials

Manage shared header and footer templates as partials. Toggle them per-page, edit with the same visual editor, and compile CSS independently.

SEO Built-In

Meta title, description, canonical URLs, robots directives, Open Graph, Twitter Cards, and JSON-LD structured data — all per-page fields.

Security First

HTML sanitization, encrypted API key storage, SVG uploads disabled by default, rate limiting on AI endpoints, and file type validation.

Page Types & Drafts

Categorize pages (blog post, landing page, etc.) with custom types. Draft/Published workflow with a designated homepage setting.

Global CSS & JS

Site-wide stylesheets and scripts managed from settings. Per-page CSS and JS fields for fine-grained control without touching your host app.

Flexible Auth

Plug in Devise, a custom lambda, HTTP Basic Auth, or a parent controller. Warns you in production if admin authentication isn't configured.

Quick Start

Up and running in
under five minutes.

Three steps to add a full page management system to your Rails app.

1

Add the gem and install

Terminal 
# Add to your Gemfile
bundle add active_canvas

# Optional: for Tailwind CSS compilation (not CDN)
bundle add tailwindcss-ruby

# Optional: for AI features (text, image, screenshot-to-code)
bundle add ruby_llm

# Run the installer
bin/rails generate active_canvas:install
bin/rails active_canvas:install:migrations
bin/rails db:migrate
2

Mount the engine

config/routes.rb 
Rails.application.routes.draw do
  # Mount at any path you like
  mount ActiveCanvas::Engine => "/canvas"

  # Or at a different path...
  # mount ActiveCanvas::Engine => "/pages"
  # mount ActiveCanvas::Engine => "/cms"

  # Or at root to power your entire site
  # mount ActiveCanvas::Engine => "/"
end
3

Configure authentication

config/initializers/active_canvas.rb 
ActiveCanvas.configure do |config|
  # Devise, a lambda, or :http_basic_auth
  config.authenticate_admin = :authenticate_user!

  # CSS framework: :tailwind, :bootstrap5, or :none
  config.css_framework = :tailwind

  # AI features (requires ruby_llm gem)
  config.enable_ai_features = true
end

That's it. You're ready.

Start your Rails server and visit your mounted paths:

Admin Panel
<mount>/admin

Create pages, manage media, configure settings, use the visual editor

Public Pages
<mount>/:slug

Published pages served to visitors with compiled CSS and SEO meta tags

AI Features

Generate content
with AI providers.

ActiveCanvas integrates with OpenAI, Anthropic, and OpenRouter via the ruby_llm gem. Configure your API keys in the admin settings panel — they're stored encrypted in your database.

Text Generation

Generate HTML content from natural language prompts. The AI respects your configured CSS framework and produces Tailwind or Bootstrap markup accordingly.

Image Generation

Create images from prompts directly inside the editor. Generated images are automatically saved to your media library via Active Storage.

Screenshot to Code

Upload a screenshot or wireframe and get clean, editable HTML + CSS back. Uses vision models to understand layouts and produce semantic markup.

Gemfile 
# Enables per-page Tailwind compilation
# (optional — falls back to CDN)
gem "tailwindcss-ruby", "~> 4.0"

# What happens on page save:
# 1. Scans page HTML for Tailwind classes
# 2. Compiles only used classes via CLI
# 3. Stores compiled CSS in the database
# 4. Serves optimized CSS per page

# Custom Tailwind config in admin
# settings panel (CSS tab):
# - Custom colors
# - Custom font families
# - Bulk recompile all pages
Tailwind CSS v4

Per-page CSS.
Not a CDN blob.

When tailwindcss-ruby is installed, ActiveCanvas compiles Tailwind CSS at save time. Each page gets only the CSS classes it actually uses — compiled via the real Tailwind CLI, not a CDN.

  • Compiled CSS stored in the database per page
  • Graceful fallback to Tailwind CDN if gem not installed
  • Custom colors and fonts via admin settings
  • One-click bulk recompile for all pages and partials
  • Also supports Bootstrap 5 and vanilla CSS

Under the Hood

What ships with the engine.

ActiveCanvas adds these models and admin interfaces to your app, all behind its own namespace.

Page

Content, slug, CSS, JS, SEO fields, draft/published

PageVersion

Diffs, change summaries, author tracking

Media

Active Storage files, thumbnails, metadata

Partial

Header/footer templates, compiled CSS

PageType

Categorize pages: blog, landing, etc.

Setting

Key-value store: homepage, CSS, API keys

AiModel

Synced from providers, toggle on/off

Admin UI

Full CRUD, settings, editor, media browser
